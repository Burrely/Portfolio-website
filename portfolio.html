Advanced Javascript

Advanced Javascript is een van de vier specialisaties van semester 2. In deze specialisatie deed ik onderzoek naar Javascript libraries, vervolgens ben ik met deze libraries aan de slag gegaan om poc’s te maken en uiteindelijk heb ik met behulp van een van deze libraries een prototype gemaakt voor mijn portfolio.

Proces

De specialisatie bestond uit drie onderdelen die waarbij we elk een week kregen.

Week 1

Ik heb een longlist van libraries ontwikkelt uit libraries die mij interessant leken. Ik heb hiervoor Google en Github gebruikt, vervolgens heb ik deze lijst verwerkt in Google Sites omdat ik destijds geen portfolio site had. Vervolgens heb ik dit ingeleverd.

Longlist

  1. Anime.js

    Anime.js is een library specifiek ontworpen voor animaties. Met de komst van CSS3 is het gebruik maken van Javascript voor animaties nogal vervreemd geworden voor front-end web developers. Maar wat als je meer kan doen met animaties? - Anime.js is hier het antwoord voor. Niet alleen kan je met Anime.js meer soorten animaties op een makkelijkere manier maken; je kan ook meer interactie creeëren tussen animaties en jouw front-end creatie. Denk aan bijvoorbeeld het live aanpassen van beweging, en de mogelijkheid om callbacks te hebben waardoor je iets kan laten gebeuren wanneer een animatie is afgerond, of bijvoorbeeld halverwege is!

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  2. Bideo

    Een heel simpele library, wat eigenlijk meer te vergelijken valt als een code snippet. Het laat je een video als background van een element gebruiken, bijvoorbeeld de body of dergelijke. Voornamelijk te gebruiken uit gemak.

    Om een voorbeeld te zien; klik hier.

  3. Chart.js

    Chart.js is een library waarmee je met gemak grafieken voor je data kan maken. De grafieken kunnen op vele manieren gevisualiseerd worden met Chart.js, ook geeft het je veel vrijheid in hoe je je grafieken wilt visualiseren. Deze library bespaart je veel tijd wanneer je veel bezig bent met data in je projecten.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  4. Cleave.js

    Cleave.js is een library voor gebruikersinput. Met deze library kan je detecteren wanneer bepaalde soort gegeven data iets is, bijvoorbeeld wat voor soort creditcard iemand gebruikt. Daarnaast kan deze library ook tekst formateren op een logische manier aan de hand van wat voor informatie er in een input veld tevoorschijn moet komen.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  5. Three.js

    Three.js is een javascript library die zich focused op het gebruik maken van 3D in de browser. Three.js maakt gebruik van WebGL en geeft de mogelijkheid aan developers om 3D elementen in te laden. Hiernaast heeft het een heleboel tools om interactie te bieden tussen de gebruiker en de 3D omgeving.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  6. Closure-library

    De Closure library is een library die Google gebruikt voor meerdere van hun web applicaties die nu open source is. Deze library geeft toegang tot UI Widgets, controls, methodes voor het aanpassen van elementen in je pagina, animaties etc. Vergelijkbaar met andere grotere libraries biedt het een ruim aantal opties ten koste van wat er allemaal geladen moet worden.

  7. Surveyjs

    Surveyjs is een library die surveys maken makkelijker maakt. Zelf vind ik het een beetje een vreemde library aangezien een survey maken met javascript niet echt bepaald heel moeilijk is. Maar het is en blijft een hulp tool zodat je het niet zelf hoeft te doen. De main selling points zijn dat het niet afhankelijk is van een andere service.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  8. Nanobar

    Nanobar is een lichte library die simpele, maar effectieve laad balken genereerd. Je kan deze laadbalken makkelijk beinvloeden doormiddel van het gebruik van meegegeven methods. Daarnaast kan je de looks aanpassen met gebruik van CSS. In het algemeen super handig als je een simpele laadbalk nodig hebt voor een van je projecten/designs.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

  9. Javascript-state-machine

    De Javascript state machine library maakt het makkelijker om met staten van een 'object' te werken, of eerder, een tekst. Zo kan je een aantal staten definieren van wat iets kan zijn en deze aanpassen met een automatisch gegenereerde methode. Je kan hier ook listeners mee laten genereren die iets kunnen doen wanneer de staat van jouw tekst/object veranderd.

  10. Google Maps - V3 utility library

    Met deze library kan je zelf je eigen applicatie maken aan de hand van Google's Google Maps service. De library geeft je de mogelijkheid deze widget te maken en er ook mee te werken. Heel handig als je iets wilt doen met geolocatie.

    Om voorbeelden te bekijken van wat de mogelijkheden zijn; klik hier.

    11. BONUS:

  11. Typescript

    TypeScript is niet echt een library, maar wel iets wat verder gaat dan standaard Javascript. TypeScript is een taal gebaseerd op Javascript, maar veranderd het in een stricte taal. - De syntax is dus precies hetzelfde, met de toevoeging dat je met types werkt in plaats van abstracte variabelen. Je kan de code die je schrijft omzetten naar gewone Javascript door gebruik te maken van een compiler. Hier bestaan tools voor in de meeste editors in de vorm van extensies. Heel interessant om iets mee te doen.

  12. Awesome-javascript

    Een collectie van javascript libraries. Leuk om mee te spelen, maar voor elke publicatie te groot omdat er zoveel onderdelen in zitten die je in het algemeen niet nodig hebt. Wel interessant om mee te spelen of doorheen te kijken als je benieuwd bent naar wat voor libraries er allemaal zijn.

Communication

Sprint 0 - Presentatie Glamorous Goat

De presentatie voor Glamorous Goat was een optioneel onderdeel voor ons project dat we hebben gedaan om onduidelijkheden over het project tijdig te verduidelijken.

Proces

We vonden de opdracht nog wat onduidelijk dus hebben we als team besloten om onderzoek te doen naar een aantal mogelijke concepten. Hieruit hadden we drie concepten gehaald die we in groepen van twee wouden gaan presenteren aan onze stakeholder; Glamorous Goat.

Concept

Ons concept was om een grijswatersysteem te ontwikkelen voor huishoudens waarmee we restwater kunnen hergebruiken, beide binnen het eigen huis en binnen een gemeente.

Voorbereiding

Na het ontwikkelen van concepten hebben we contact opgenomen met Glamorous Goat om een afspraak te maken voor onze pitches. Na een bevestiging te hebben gekregen zijn we begonnen met het uitwerken van onze pitches hebben we een middag geoefend, daarna hebben we onze pitches aan elkaar gepresenteerd en feedback op elkaar gegeven. Op de dag van de pitch hebben we dit nog een keer gedaan terwijl we de feedback meegenomen hadden ter voorbereiding.

De pitches

De pitches zelf verliepen soepel. We hadden kleine problemen met de hardware waarmee we de pitches opgenomen hadden ondanks we deze van tevoren hadden gechecked, maar uiteindelijk verliep het toch allemaal soepel. We hebben ook al onze feedback gedocumenteerd.

Conclusie

Het is ons gelukt om een antwoord te krijgen op onze vraag; Glamorous Goat wou een applicatie design voor een water meetsysteem. Geen ander concept ondanks ze onze concepten wel interessant vonden.

Leeruitkomsten

  • Communication

    We hebben met onze stakeholders contact genomen voor duidelijkheid over onze opdracht, dit aan de hand van een presentatie die we hebben gedocumenteerd met video en audio. Daarnaast hebben we de feedback gedocumenteerd.

Reflectie

Wat ik geleerd heb van dit project is dat je vaak dingen over het hoofd ziet ondanks je denkt dat je wel goed zit. Meerdere iteraties en het herhaaldelijk vragen van feedback heeft geleid tot een zeer sterk product, in dit geval een pitch.
